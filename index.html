Transitions

A transition is a change of state from an initial to an end state. The transition style has two properties. The first defines which style is the transition and the other property/parameter defines the duration of the transition.


In this demo, when you hover over the button, the object will change from a small dark red button to a larger button, with a white background. I have given the button the styling for this effect by using the :hover pseudo class.

transition: property duration; - The duration can be measured in milliseconds or seconds.

transition: all 2s

Above I selected the all option property and a duration of 2s to see the transition take place.

CSS | Transition (Shapes) Hover Effect


In the example above, I created a small box with a position relative and a applied styling to the left, bottom, color, width and height properties. On the hover event, I changed the styling and also the border-radius. Previously, on the hover state of the object, I used the 'all' option, but in this demo I didn't. Using 'all' can affect the browsers performance, if it is tracking a range of styles. As a result, you can also place specific styles in a comma separated list of property name and duration.

transition: property-name duration, property-name duration;

transition: left 4s, width 4s, height 4s, background-color 2s;

Managing Transition Timing

The timing of transition is controlled by adding a timing value to the transition property.

transition: property duration timing;

Timing can be defined by using 5 key words:

  • ease - style changes slowly, speeds up in the middle and ends slowly [default].
  • linear - maintains constant rate throughout duration.
  • ease-in - starts slow and speeds up to the end.
  • ease-out - starts fast and slows down to the end
  • ease-in-out - starts slower than ease timing, speeds up and then slows down towards the end.

Managing Transition Timing Delay

transition: property duration timing delay;

transition: width 2s ease 0s, height 2s ease 0s, left 1s ease 2s, background-color 2s ease 4s;

Why delay a transition? We usually do this when we have several styles that are changing and want to apply them in a sequential order.
